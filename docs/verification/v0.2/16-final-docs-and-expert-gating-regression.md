```text

```

`/solo` 和 `/solo-spec` 只作为宿主环境已配置别名时的兼容写法，不再作为推荐入口。

## 4. 专家增强规则

最终规则：

- 默认不自动调用专家。
- 到达可增强阶段的用户门禁前，主流程必须报告当前阶段专家增强状态。
- 只报告当前阶段映射专家，不枚举全部已安装 Skill。
- 已安装当前阶段专家时，用户可选择调用该专家或跳过专家并确认当前阶段。
- 未安装当前阶段专家时，用户可跳过专家或指定其他 Skill / 工具审查当前阶段。
- 专家输出仍由主流程消费，专家不写文件、不改状态机、不通过门禁。

## 5. 回归发现与修复

真实黑盒复测发现：`basic` 安装进入 `brainstorm` 后，主流程曾只输出“未调用专家”，没有说明 `$solo-spec-product` 是否安装，也没有给出“跳过 / 指定其他 Skill”的选择。

修复：

- 加固 `skills/solo-spec/SKILL.md`。
- 同步 `skills/solo-spec/references/experts.md`。
- 要求 mapped stage 的 `专家增强` 必须写出映射专家、检测状态、使用状态和下一步选择。
